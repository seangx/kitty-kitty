核心概念

Room 是一等公民

灵感来自游戏大厅的 Room 模型和 Matrix 协议。每个协作场景对应一个 Room，Room 是所有交互的容器。不存在脱离 Room 的消息或任务。

Room 包含的元素

Messages
对话消息
Tasks
结构化任务
Artifacts
产出物
File Leases
文件租约
Summaries
压缩摘要

Room 类型

dm
私聊

两个 Agent 间的 1:1 通道

team
团队频道

持久化的团队协作空间

task
任务房间

围绕一个具体任务的短生命周期房间

project
项目房间

长期项目空间，可包含子 Room

lobby
大厅

默认全局空间，所有 Agent 自动加入

设计原则
Append-only Timeline

一切都是 Room 时间线上的 Event。不可变、可追溯、可压缩。

Task 状态机

Task 通过 Room Event 驱动状态流转。每次状态变更对应一个 task-* 类型事件。

submitted
— claim →
working
— complete →
completed
working
— ask →
input-required
— answer →
working
working
— fail →
failed
任何非终态
— cancel →
canceled
终态

completed failed canceled

进入终态后不可再转换。

人机交互循环

working ↔ input-required 循环可多次往返，用于 Agent 向委托人请求澄清或额外输入。

05

三层消息检索

L1 增量查询 events(since=offset) — 日常使用，零成本。Agent 记住上次 offset，只拉新 events。
L2 摘要 + 最近 N 条 新成员加入时，LLM 生成 summary state event，后续查询返回摘要 + 摘要之后的新 events。
L3 全量归档 SQLite 完整保留所有 events，按需分页拉取。用于审计和回溯。

自动压缩触发

1 Room events 累积
→
2 events > 100 条
→
3 LLM 生成 summary event
→
4 后续查询 = summary + 新 events

Summary event 本身也是一个 room event（type=summary），包含之前所有 events 的结构化摘要。原始 events 保留在 DB 中不删除。

06

File Lease（文件租约）

借鉴 MCP Agent Mail 的设计。Agent 声明对文件的修改意图，其他 Agent 可查询冲突。

协商性质

Advisory, not blocking — 租约不会硬锁文件，而是表面化冲突。Agent 自行决定是否继续。人类团队中的"我在改这个文件"口头约定的数字化。

生命周期

租约有过期时间。Agent 可主动 release，也可 renew 续期。过期后自动释放。可选的 pre-commit guard 在提交前检查租约归属。

冲突发现流程

1 Agent A: lease.claim
→
2 file-lease event 写入 Room
3 Agent B: lease.check
→
4 发现冲突：Agent A 持有
→
5 协调（DM / 等待 / 继续） 
// 示例：Agent 声明正在修改认证模块
hive.lease.claim(roomId, {
  paths: ["src/auth/*.ts", "src/middleware/auth.ts"],
  reason: "重构认证中间件，添加 OAuth2 支持",
  duration: "30m"
});
07

MCP 工具层

所有工具通过 MCP stdio 协议暴露，遵循 JSON-RPC 2.0 规范。

Room 管理
hive.rooms.list(filter?)              // 列出房间（可按 kind/visibility 过滤）
hive.rooms.find(query)               // FTS5 全文搜索房间及内容
hive.rooms.create({name, kind,       // 创建房间
  members?, visibility?})
hive.rooms.open(room_url_or_id)      // 打开/加入房间
hive.rooms.close(room_id)            // 关闭房间
hive.rooms.info(room_id)             // 获取房间详情
Room 交互
hive.room.post(room_id,              // 发送事件到房间
  {type, content?, task?, artifact?})
hive.room.events(room_id, since?)    // 增量拉取事件流
hive.room.watch(room_id)             // 订阅推送（SSE / polling）
hive.room.members(room_id)           // 查看房间成员
hive.room.summary(room_id)           // 触发/获取摘要
File Lease
hive.lease.claim(room_id,            // 声明文件修改意图
  {paths, reason, duration?})
hive.lease.check(paths)              // 查询路径冲突
hive.lease.release(lease_id)         // 主动释放租约
hive.lease.renew(lease_id, duration?)// 续期租约
糖衣 API（Macro）
hive.start()                         // 注册 agent + 加入 lobby + 拉 inbox
                                      // = macro_start_session
hive.task(to, title, input?)          // 自动建 task room + post task-start
hive.dm(to, content)                  // 自动找/建 dm room + post message
hive.broadcast(group, content)        // post 到 team/lobby
hive.check(task_id)                   // 快速查 task 状态
设计理念

底层 API（rooms.*、room.*、lease.*）提供完整控制力。糖衣 API（hive.*）用最少调用完成最常见操作。Agent 可根据需求选择使用哪一层。

09

SQLite 并发架构

借鉴 MCP Agent Mail 经过实战验证的模式。

WAL 模式

Write-Ahead Logging 模式 + NORMAL 同步级别。读写并发，不阻塞读操作。

连接池

保守连接限制。每个连接返回池时执行 passive WAL checkpoint。

锁竞争处理

指数退避 + 抖动（jitter），避免 lock contention 下的惊群效应。

FTS5 自动索引

通过 SQLite 触发器自动维护全文搜索索引，零额外写入成本。

熔断器状态机

Circuit Breaker 保护数据库免受级联故障影响。

closed
正常工作
— N 次失败 →
open
熔断中
— 超时 →
half-open
试探中
half-open
— 成功 →
closed
// 并发配置概览
{
  "journal_mode": "WAL",
  "synchronous": "NORMAL",
  "busy_timeout": 5000,
  "max_connections": 3,
  "circuit_breaker": {
    "failure_threshold": 5,
    "reset_timeout_ms": 30000
  },
  "backoff": {
    "initial_ms": 50,
    "max_ms": 2000,
    "jitter": true
  }
}

Matchmaking（撮合）

1 Agent A 创建 task room
→
2 声明 required_skills
→
3 Matchmaker 扫描 agents 表
4 按 role/expertise 匹配
→
5 发送 Room 邀请
→
6 候选人 accept → join → task 开始
本地撮合

扫描本机 agents 表中 status=idle 且 roles/expertise 匹配的 Agent。O(n) 扫描，n 通常 < 20。

联邦撮合（Phase 2+）

本地无匹配时，通过 gateway 向联邦节点广播能力请求。远端 hive 返回候选列表。

  • MCP stdio 作为唯一对外接口（Phase 1）
